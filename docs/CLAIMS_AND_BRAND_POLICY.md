# Claims and Brand Policy

## Rule

Do not copy competitor wording, logos, screenshots, documentation structure, or pricing-page language.

Do not claim compatibility with another product unless VEDA Runtime has a tested adapter, documented supported versions, and a passing integration test.

## Approved Claim Style

Use evidence-based wording:

- "Runs a local proof pipeline with build, tests, demos, and status output."
- "Uses HMAC chaining for tamper-evident local audit records."
- "Blocks unsafe shell commands by default through the sandbox policy."
- "Writes machine-checkable pipeline artifacts under `logs/`."

## Avoid

Do not say:

- "Guaranteed to prevent all AI damage."
- "Enterprise compliant" without a real compliance review.
- "Drop-in replacement" for any named product.
- "Compatible with" another product unless the tested adapter exists.
- "SOC 2 ready" or similar compliance language without the actual program.

## Competitive References

If a customer asks how VEDA Runtime compares to another tool, answer with factual category differences only. Do not use competitor names in package keywords, ad copy, logos, badges, docs, or metadata unless legal review approves it.

## Evidence Rule

A product claim is allowed only when it points to one of:

- a passing test
- a passing `npm run release:check` artifact
- a working demo command
- a documented source file in this repository
